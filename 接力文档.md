# 税务自动化系统 - 接力文档

## 1. 项目概述

**项目名称：** 税务自动化系统 (Tax Automation System)  
**技术栈：** Python 3.10+ / Flask / SQLite / Playwright  
**功能：** 企业管理、记账管理、税务计算、纳税申报、天眼查自动查询

## 3. 部署指南

### 3.1 环境要求
- Ubuntu 22.04 / Windows 10+
- Python 3.10+
- Playwright + Chromium

### 3.2 首次部署

```bash
# 1. 克隆仓库
git clone https://github.com/ljh010704/tax-automation.git
cd tax-automation

# 2. 安装依赖
pip install -r requirements_web.txt

# 3. 安装 Playwright 浏览器
python3 -m playwright install chromium

# 4. 创建 .env（必须设置 SECRET_KEY）
echo "SECRET_KEY=$(python3 -c 'import secrets; print(secrets.token_hex(32))')" > .env

# 5. 启动
PORT=8080 python3 run_web.py
```

### 3.3 日常更新

```bash
cd /opt/tax-automation
git pull origin main
PORT=8080 python3 run_web.py
```

## 5. 核心功能说明

### 5.1 用户认证
- 注册/登录/退出
- 密码使用 Werkzeug PBKDF2-SHA256 哈希
- 所有 POST 接口有 CSRF 防护

### 5.2 企业管理
- 添加/编辑/删除企业
- 天眼查自动查询（输入 18 位信用代码自动填充）
- 多用户数据隔离（user_id 过滤）

### 5.3 记账管理
- 收入/支出记录
- 月度/季度汇总
- 报表统计

### 5.4 税务计算
- 增值税（小规模/一般纳税人）
- 印花税
- 社保
- 个人所得税（年度汇算清缴）

## 6. 安全审计修复记录

| 问题 | 修复 |
|------|------|
| 多用户数据隔离不完整 | update_entity/delete_transaction 增加 user_id 过滤 |
| 缺少 CSRF 防护 | 新增 csrf.py，所有 POST 路由加 @csrf_required |
| 默认密钥可用于生产 | 无 SECRET_KEY 时拒绝启动 |
| 密码哈希过弱 | SHA-256 → Werkzeug PBKDF2-SHA256 |
| 季度计算崩溃 | 补上 quarterly_expenses 参数 |
| 配置项未生效 | 支持 DATABASE_PATH/WEB_DATABASE_PATH 环境变量 |
| 输入校验不足 | 金额不允许负数/NaN/无穷大 |
| 调试接口暴露 | 删除 /entities/api/debug |
| 错误信息暴露内部异常 | 记录日志 + 用户友好提示 |

## 7. 常见问题

### Q: 查询按钮点不动？
- 浏览器缓存旧 JS → Ctrl+Shift+R 强制刷新
- 服务器未更新 → git pull + 重启

### Q: 查询返回 503？
- 未安装 Playwright → pip install playwright && python3 -m playwright install chromium

### Q: 登录地址/省份为空？
- 天眼查页面结构变化导致提取失败
- 检查 core/business_query.py 中的 CITY_PROVINCE_MAP 反查逻辑

### Q: 端口被占用？
```bash
lsof -i :8080
kill -9 <PID>
```
